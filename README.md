# 📨 Telegram Aggregator

Userbot который пересылает сообщения из нужных каналов в каналы-агрегаторы.
Поддерживает несколько групп — каждая группа имеет свои источники и свой канал назначения.

---

## 🚀 Установка и запуск (пошагово)

### Шаг 1 — Получи Telegram API ключи

1. Зайди на [my.telegram.org](https://my.telegram.org) **под своим вторым аккаунтом**
2. Нажми **"API development tools"**
3. Заполни форму (App title и Short name — можно написать что угодно, например "MyAggregator")
4. Platform — выбери **Other**
5. Скопируй **App api_id** (число) и **App api_hash** (длинная строка)

---

### Шаг 2 — Узнай ID каналов-агрегаторов

Для каждого канала куда будут пересылаться сообщения:

1. Создай канал в Telegram (или используй существующий)
2. Сделай **второй аккаунт администратором** этого канала
3. Перешли любое сообщение из канала боту [@userinfobot](https://t.me/userinfobot)
4. Он пришлёт ID — число вида `-1001234567890`
5. Скопируй это число — оно понадобится в `config.py`

---

### Шаг 3 — Настрой проект

```bash
# Установи зависимости
pip install -r requirements.txt

# Скопируй файл с примером настроек
cp .env.example .env
```

Открой файл `.env` и заполни своими данными:
```
API_ID=12345678           ← вставь свой api_id
API_HASH=abcdef...        ← вставь свой api_hash
PHONE_NUMBER=+79991234567 ← номер телефона аккаунта
```

---

### Шаг 4 — Настрой группы каналов

Открой `config.py` и опиши свои группы.
Каждая группа — это список каналов-источников и канал-агрегатор куда их пересылать:

```python
CHANNEL_GROUPS = [
    {
        "name": "Крипта",                    # название группы (для удобства)
        "sources": [
            "@crypto_channel1",
            "@crypto_channel2",
        ],
        "aggregator_id": -1001234567890,     # ID канала-агрегатора для этой группы
    },
    {
        "name": "Новости",
        "sources": [
            "@news_channel1",
            "@news_channel2",
        ],
        "aggregator_id": -1009876543210,     # ID другого канала-агрегатора
    },
    # Можно добавлять сколько угодно групп
]
```

⚠️ Убедись что второй аккаунт **подписан** на все каналы-источники, иначе бот не будет их видеть.

---

### Шаг 5 — Запусти!

```bash
python main.py
```

**При первом запуске:**
- Введи номер телефона (если спросит)
- Введи код из Telegram
- Готово! Сессия сохранится в файл `aggregator_session.session`

При следующих запусках — авторизация не нужна.

## ❓ Частые вопросы

**Бот запущен но ничего не пересылает — что делать?**
Скорее всего второй аккаунт не подписан на каналы-источники.
Зайди в Telegram под вторым аккаунтом и вручную подпишись на все каналы из `config.py`.

**Программа упала с ошибкой — что делать?**
Посмотри текст ошибки в консоли. Чаще всего причина:
- Неверный API_ID или API_HASH → перепроверь `.env`
- Неверный ID канала → перепроверь `aggregator_id` в `config.py`
- Второй аккаунт не является администратором канала-агрегатора

**Как сбросить сессию и залогиниться заново?**
```bash
rm aggregator_session.session
python main.py
```

**Как остановить бота?**
Нажми `Ctrl+C` в консоли.

**Файл aggregator_session.session — что это?**
Сохранённая сессия. Не удаляй его без необходимости, иначе придётся логиниться заново.
